# Epic 5 — AI-сгенерированные insight-карточки (paste-from-Claude flow)

**Дата:** 2026-05-13
**Базовый research:** [`2026-05-13-audio-insights-research.md`](./2026-05-13-audio-insights-research.md)
**Predecessor:** Epic 4 (audio → transcript) — закрыт.

---

## Цель

Тренер не создаёт карточки руками. Вместо этого:

1. После транскрипции скачивает `.txt`
2. В своём чате с Claude (claude.ai Pro/Max — без API-биллинга) запускает свой промпт
3. Копирует ответ Claude обратно в Nivel
4. Парсер превращает его в **draft-карточки** на странице сессии
5. Тренер свайпает: ✓ approve / ✎ edit / ✗ reject
6. Approved карточки видит ученик

**Главное отличие от Epic 4:** ручное создание карточек **удаляется**. Карточки появляются только через paste-from-Claude.

---

## Утверждённые решения

| Вопрос | Решение |
|--------|---------|
| Когда генерим инсайты? | Не автоматически — тренер сам идёт в claude.ai когда захочет |
| Где живут draft-карточки? | На `/sessions/[id]` среди финальных, маркер «AI draft» |
| Можно ли создавать карточки руками? | **Нет.** Удаляем все manual creation UI |
| Категории/теги | Канонические: техника / тактика / физика / ментал |
| Цитата из транскрипта | Обязательна — якорь против галлюцинаций |
| Повторный paste | Заменяет существующие draft (старые draft удаляются, approved/rejected остаются) |
| Свайп-ревью | На `/sessions/[id]` (там же где карточки) |
| Что видит ученик | Только approved карточки |
| Промпт для Claude | **НЕТ кнопки «копировать промпт»** — тренер пишет свой |
| Подсказка про формат | Маленький `<details>` под textarea — необходимо для работы парсера |

## Формат для парсера

```
## Карточка 1
- Тема: техника
- Заголовок: Поздний замах при бандеха
- Описание: При бандеха ты начинаешь замах когда мяч уже на уровне плеча. Начинай замах когда мяч ещё летит к тебе — на 0.5 сек раньше.
- Цитата: "ты опять поздно взял, видишь — мяч уже тут а ты только начинаешь"

## Карточка 2
- Тема: тактика
...
```

**Правила парсера:**
- Каждая карточка начинается с `## `
- 4 обязательных поля: Тема / Заголовок / Описание / Цитата
- Тема ∈ {техника, тактика, физика, ментал} (case-insensitive)
- Если поле отсутствует или Тема не валидна → парсер показывает ошибку, не создаёт карточки
- Цитата может быть в кавычках или без — парсер тримит

## Что удаляем из существующего UI

1. **`InlineSessionCardAdder`** на странице сессии — удалить компонент и его вызов
2. **«Создать инсайт из выделения»** на `/sessions/[id]/transcript` (фича M6 из Epic 4) — удалить bottom-bar и модалку `CreateInsightModal`
3. **`+Insight` кнопки** в любом другом месте (dashboard ученика для тренера, и т.д.)

Существующие manual карточки (`source='manual'` или без source) **остаются** в БД — но новых нельзя создать.

## Что добавляем

### Страница транскрипта `/sessions/[id]/transcript`
- Кнопка **«Скачать транскрипт»** — генерит `.txt` (сплошной текст из `raw_text`) и скачивает

### Страница сессии `/sessions/[id]`
- Кнопка **«Вставить инсайты»** (для тренера) — открывает bottom-sheet с textarea
- В модалке:
  - Textarea для paste
  - `<details>` «Ожидаемый формат» с примером 1-2 карточек
  - Кнопка «Создать карточки»
  - При ошибке парсинга — показывает что не так (например «строка 12: неизвестная Тема»)
- После успешного парса — модалка закрывается, на сессии появляются draft-карточки

### Карточки на странице сессии
- Draft-карточки визуально отличаются от approved (значок «AI», другой цвет рамки)
- Над draft-карточкой 3 кнопки:
  - **✓ Approve** — `trainer_status = 'approved'`, ученик видит
  - **✎ Edit** — открывает модалку редактирования (title/body/tags, **не** quote)
  - **✗ Reject** — `trainer_status = 'rejected'`, скрывается
- Approved карточки — обычные (как сейчас)

## БД

Изменения схемы **не нужны**. `insight_cards` уже имеет:
- `source` — будем писать `'ai-paste'` для новых
- `trainer_status` — `draft` / `approved` / `rejected`
- `quote` — текст цитаты из транскрипта
- `tags` (JSONB) — массив тегов

## Workflow (правильный, после фиксапа [[epic-pr-summary]])

1. ✅ Epic-branch `epic/5-ai-insights` создан из main и запушен
2. ✅ Draft epic-PR `epic/5-ai-insights → main` (этот документ + ссылка на план)
3. Child issues создаются с инструкцией `--base epic/5-ai-insights` для `gh pr create`
4. Routine-prompt агентов содержит `gh pr create --base epic/5-ai-insights`
5. Каждый task-PR мерджится в epic-branch (не в main)
6. Когда все task-PR смерджены — epic-PR переводится из draft в ready
7. Финальный summary-комментарий на epic-PR
8. Merge epic-PR в main одним squash-коммитом

## Acceptance (целиком эпика)

1. Тренер заходит на `/sessions/[id]/transcript` → нажимает «Скачать транскрипт» → получает `.txt`
2. Идёт в claude.ai, прикрепляет файл, пишет свой промпт, получает markdown
3. Возвращается в Nivel → `/sessions/[id]` → нажимает «Вставить инсайты»
4. Пастит markdown → видит draft-карточки на сессии
5. Свайпает: одни approve, одни reject, одну редактирует
6. Ученик открывает свой dashboard → видит только approved карточки этой сессии
7. **Невозможно** создать карточку никаким другим способом
