# Mobile-first

Nivel — **mobile-first** сервис. Любая страница, компонент, модалка, форма проектируется под мобильный viewport (∼390px) в первую очередь. Десктоп — производное.

- Контейнеры ограничивай `max-w-[430px]`, центрируй.
- Тач-зоны ≥ 44px, не полагайся на `:hover`.
- Модалки/sheet'ы используй bottom-sheet pattern (через portal, чтобы не застревать в `transform`-контексте родителя). Не открывай обычные центрированные диалоги.
- Длинные горизонтальные списки (карусели) делай с `overflow-x-auto` + `snap-x snap-mandatory`, не вертикальный stack.
- Шрифты, отступы, иконки выбирай под палец, не курсор. Проверяй на узком экране до того как зафиксировать дизайн.

# План — гайдлайн, acceptance — закон

Планы и issue пишутся в момент X, ты выполняешь в момент Y. За это время другие эпики могли изменить те же файлы, переименовать поля, поменять схему БД. **Не пиши код по плану слепо.**

**Перед началом реализации:**

1. **Прочитай реальное состояние** всех файлов, упомянутых в issue (любые пути `src/...`, `supabase/...`, `docs/...` в кодовых блоках или в тексте). Используй `Read`/`cat`, не доверяй памяти.
2. **Сравни с тем что описано в issue.** Три ситуации:
   - **Совпадение** → пиши по плану.
   - **Мелкие расхождения** (поле переименовано, импорт переехал, появился новый параметр, изменилась сигнатура соседней функции) → **адаптируй** код под текущее состояние, сохрани цель.
   - **Большие расхождения** (компонент уже сделан кем-то другим, файл удалён, схема БД отличается от плана, нужная функция уже существует под другим именем) → реализуй **цель из секции Acceptance**, опираясь на реальный код, а не на устаревший план.
3. **Не блокируй** задачу из-за устаревшего плана. Блок — только если acceptance физически невыполним (нужного API нет в стеке, требуется доступ которого у тебя нет, противоречивые требования).
4. **В PR-описании** отдельным блоком отметь если адаптировал: `## Отклонения от плана` — что и почему. Это помогает ревью.

**Главное правило:** ты отвечаешь за то чтобы acceptance работал на реальном репозитории. План — это снимок чужой головы в прошлом. Acceptance — это контракт.
